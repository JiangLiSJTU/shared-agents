# Markdown to Executive Word Formatter

This skill is designed for developing Python scripts that parse Markdown files and generate `.docx` documents matching strict formatting standards for technical or executive reports.

## 1. Environment Requirements

When creating the Python script:
*   Use `python-docx` (`pip install python-docx`). Do NOT use pandoc unless specifically requested, as it lacks fine-grained paragraph-level control for the "bold-red" rule.
*   The script should accept a target `.md` path and output a `.docx` path.

## 2. Core Formatting Specs (Must Implement)

The script generated by this skill must incorporate the following logic:

### A. Typography and Indentation
*   **Fonts**:
    *   Chinese Characters: **SimSun** (宋体)
    *   English Characters/Numbers: **Arial**
*   **Indentation**: Regular paragraphs must have a first-line indent of `0.35 inches` (approx. 2 characters).
*   **Spacing**: Ensure appropriate space after paragraphs (e.g., `Pt(6)`).
*   **Headings**: Provide clear visual distinction (e.g., Heading 1: 18pt, Heading 2: 16pt, Heading 3: 14pt). Ensure "Keep with next" is set to `True` for all headings to prevent widow/orphan issues.

### B. The "Highlighter Rule" (Bold & Red)

When parsing inline Markdown boldness (`**bold**`):
1.  **If the bold text contains ONLY critical performance metrics** (i.e., numbers combined with units, like `20%`, `50 TOPS`, `99.3 TFLOPS/W`, `12-18个月`), format the run as `font.bold = True` AND `font.color.rgb = RGBColor(255, 0, 0)` (Red).
2.  **If the bold text contains general concepts or phrases**, format the run as `font.bold = True` but DO NOT change the color (keep default black).
3.  *Note: A regex-based or rule-based parser within the script must distinguish these two states perfectly.*

### C. Images and Tables
*   **Images**: Parse `![caption](path)` blocks.
    *   Images must be horizontally centered.
    *   Set image width to fit the page gracefully (e.g., `Inches(6.0)`).
    *   The paragraph immediately preceding an image MUST have `keep_with_next = True` to avoid leaving large blank areas if an image drops to the next page.
    *   The caption should be placed immediately below the image, centered, with a slightly smaller font (e.g., 10pt).
*   **Tables**: Parse markdown tables (`| col1 | col2 |`).
    *   Use the `Table Grid` or a clean, professional style (e.g., `Light Shading`).
    *   Auto-fit columns where possible.
    *   Center the table on the page.

### D. Citations
*   Ensure that numeric citations (e.g., `[1]`, `[12]`) are rendered cleanly without trailing or leading markdown escape characters like `\`. Ensure they adopt superscript formatting if required by the user, otherwise keep them standard.
